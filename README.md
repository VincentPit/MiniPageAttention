# MiniPageAttention

A teaching-scale reimplementation of the [PagedAttention](https://arxiv.org/abs/2309.06180)
runtime — block-paged KV cache, refcounted prefix sharing, iteration-level
("continuous") batching, and a paged-decode Triton kernel modelled on vLLM's
`paged_attention_v2` launcher. Built around GPT-2 + HuggingFace `transformers`
so the whole thing runs on a laptop CPU (slowly) or a single Colab GPU.

It accompanies the report in [`doc/report.tex`](doc/report.tex), *Three Layers of
LLM Inference: A Critical Reading of FlashAttention, Orca, and PagedAttention* —
the code exists to ground the report's critique in something that actually has to
confront the integration questions the original papers leave informal.

## What's here

| Component | File | Role |
|---|---|---|
| Block pool | `paged_cache.py` (`BlockManager`) | Global pool of fixed-size KV blocks shared across layers/sequences; refcounting + copy-on-write. |
| Paged cache (v1) | `paged_cache.py` (`PagedCache`) | A HF `Cache` backed by the block pool. Uniform-batch only; gathers to `[B,H,T,D]` before stock attention — preserves PagedAttention's *allocation* behaviour without a custom kernel. |
| Paged cache (continuous) | `continuous_cache.py` (`ContinuousPagedCache`) | Variable-length, dynamic-membership cache: slots activate/deactivate, active set is what the next forward operates on. |
| Uniform scheduler (v1) | `cpu/scheduler.py` (`Scheduler`) | Runs a request list as one lockstep decoding batch; records TTFT / latency / output tokens. |
| Continuous scheduler (v2) | `cpu/continuous_scheduler.py` (`ContinuousScheduler`) | Iteration-level scheduling: one prefill call admits a queued request, one decode call advances the running batch, per step. |
| Paged-decode kernel | `gpu/paged_attention_kernel.py` | `paged_attention_decode_{torch,triton}` — attention computed directly from paged memory (no gather). Triton single-pass online softmax; torch backend is the CPU sanity path. |
| GPT-2 patch | `gpu/paged_attn_patch.py` | Monkey-patches `GPT2Attention` to route decode steps through the paged kernel; prefill and non-paged caches fall through to stock HF. |
| Workloads / metrics | `bench/` | `Request` streams (synthetic lognormal, ShareGPT replay, shared-prefix, fixed-length) + a no-model allocate/free simulator for fast fragmentation sweeps. |
| Serving | `serving/api.py` | Minimal FastAPI wrapper around `ContinuousScheduler` (synchronous batch endpoint). |
| Report | `doc/report.tex` | The write-up. Figures live in `plots/`, raw run logs in `results/`. |

## Layout

```
paged_cache.py            BlockManager + PagedCache (v1)
continuous_cache.py       ContinuousPagedCache (variable-length)
cpu/
  scheduler.py            uniform-batch scheduler (v1)
  continuous_scheduler.py iteration-level scheduler (v2)
  experiments/            E1–E5 + plot_* scripts
gpu/
  paged_attention_kernel.py  torch + Triton paged-decode kernels
  paged_attn_patch.py        GPT-2 attention monkey-patch
  run_all.py                 one-shot GPU runner (E1, E3, E5, E6 + figures)
  test_paged_kernel.py       synthetic kernel correctness harness
bench/                    workload generators + measurement / simulator
serving/api.py            FastAPI server
plots/                    report figures (regenerated by run_all / plot_*)
results/                  raw run logs
doc/report.tex            the report
colab_run.ipynb           Colab driver notebook
```

## Setup

```bash
pip install torch transformers numpy            # core
pip install matplotlib datasets                 # experiments / figures (datasets = real-length workloads)
pip install triton                              # GPU paged-decode kernel (CUDA only)
pip install fastapi uvicorn pydantic            # serving/api.py only
```

Python 3.10+ and `transformers >= 5.0` (the `Cache`-as-container-of-layers API).
Everything is device-agnostic; with no GPU it runs on CPU + fp32 (the Triton
backend simply isn't used).

## Running things

All commands are run from the repo root.

```bash
# Experiments (CPU, GPT-2 small)
python -m cpu.experiments.e1_correctness          # PagedCache vs DynamicCache, B=1 greedy
python -m cpu.experiments.e2_fragmentation        # simulator: paged vs naive peak block usage
python -m cpu.experiments.e3_prefix_sharing       # N parallel completions sharing one prompt
python -m cpu.experiments.e4_throughput           # batched-correctness + tokens/sec sweep
python -m cpu.experiments.e5_continuous_vs_uniform  # v2 vs v1 on a mixed-length workload

# Figures -> plots/
python -m cpu.experiments.plot_e2
python -m cpu.experiments.plot_e2_real            # streams real conversation lengths (needs `datasets`)
python -m cpu.experiments.plot_e3
python -m cpu.experiments.plot_e5_sweep

# Kernel correctness (uses Triton if CUDA is available, else torch reference)
python -m gpu.test_paged_kernel

# One-shot GPU run: E1, E3, E5, E6 (kernel parity) + regenerates all four report figures
python -m gpu.run_all                             # falls back to CPU+fp32 with a warning if no GPU

# Serving
uvicorn serving.api:app --host 0.0.0.0 --port 8000
curl -s -X POST http://localhost:8000/v1/completions \
  -H 'Content-Type: application/json' \
  -d '{"prompts": ["The capital of France is", "Hello"], "max_tokens": 8}'
```

On Colab: `pip install -q torch transformers numpy triton matplotlib datasets`,
upload the project folder, then `cd MiniPageAttention && python -m gpu.run_all`.
See `colab_run.ipynb`.

## Experiments at a glance

- **E1 — correctness.** Paged path is greedy-identical to HF `DynamicCache` (B=1).
- **E2 — fragmentation.** Naive per-slot `max_seq_len` allocation vs paged: ~4.5× peak-occupancy gap on a synthetic ShareGPT-shaped workload, ~5.6× on real conversation lengths.
- **E3 — prefix sharing.** 8 parallel completions of a 256-token prompt: ~8× prefill memory saving (the y=N bound), ~5.7× end-to-end (output blocks stay per-sequence).
- **E5 — continuous vs uniform.** v2 ≈1.3× faster than a uniform static batch on a 16-request mixed-length workload; the advantage widens with prompt-length variance.
- **E6 — kernel parity.** The paged-decode kernel reproduces the gather-then-attention path token-for-token (modulo floating-point reduction order on near-tied logits).

## Scope / non-goals

Deliberately omitted, since they obscure the core access pattern without changing
it: FP8 KV cache, ALiBi, block-sparse attention, GQA scales, tensor parallelism,
the v2 partition+reduce split, chunked prefill, and async multi-tenant serving.
The report's "v2 launcher" section maps exactly which vLLM machinery is and isn't
mirrored here.
